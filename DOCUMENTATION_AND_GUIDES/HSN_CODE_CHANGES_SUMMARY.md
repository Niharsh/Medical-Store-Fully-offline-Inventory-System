**Added HSN Model (Lines ~35-63):**
```python
class HSN(models.Model):
    """HSN (Harmonized System Nomenclature) Codes for GST classification"""
    hsn_code = CharField(max_length=20, unique=True, primary_key=True)
    description = CharField(max_length=255, blank=True)
    gst_rate = DecimalField(max_digits=5, decimal_places=2)
    is_active = BooleanField(default=True)
    created_at = DateTimeField(auto_now_add=True)
    updated_at = DateTimeField(auto_now=True)
    
    class Meta:
        ordering = ['hsn_code']
        indexes = [
            models.Index(fields=['hsn_code']),
            models.Index(fields=['gst_rate']),
        ]
```

**Modified Product Model:**
- Added `hsn` ForeignKey field linking to HSN model
- Allows null/blank for backward compatibility
- Related name: 'products'

**InvoiceItem Model** (Already had `hsn_code` field):
- Already supports storing HSN code as string
- Used for storing snapshot of HSN at time of invoice creation

class HSNViewSet(viewsets.ModelViewSet):
    """HSN Codes - manage HSN codes for GST classification"""
    queryset = HSN.objects.filter(is_active=True)
    serializer_class = HSNSerializer

## 🔄 Key Integration Points

### 1. Product-HSN Linking
- **Model Level**: Product has ForeignKey to HSN
- **Form Level**: User selects HSN from dropdown
- **API Level**: ProductSerializer returns hsn_code and gst_rate

### 2. Billing Auto-Fill
- **Service Level**: Product API returns HSN details
- **Form Level**: handleItemChange() auto-fills HSN and GST
- **Backend Level**: InvoiceCreateSerializer uses product.hsn during creation

### 3. Invoice Storage
- **InvoiceItem**: Stores snapshot of HSN code and GST rate
- **Snapshot Pattern**: Changes to product HSN don't affect existing invoices
- **Backward Compatible**: Null HSN codes handled gracefully

### 4. Display & Calculation
- **Invoice Print**: Shows HSN in items table
- **GST Calculation**: Uses item-level gst_percent (from HSN or user input)
- **CGST/SGST**: Split 50-50 automatically

## 🎓 Key Implementation Decisions

1. **HSN as Primary Key**: Ensures uniqueness and simplifies lookups
2. **ForeignKey on Product**: Links products to HSN master
3. **Snapshot in InvoiceItem**: Preserves HSN/GST at time of invoice creation
4. **Auto-Fill in Billing**: Reduces manual entry and errors
5. **Backward Compatible**: Existing products/invoices unaffected
6. **Service Pattern**: Consistent with existing ProductType pattern
